概览：方案与能力

我们用静态站点生成器（推荐 Astro 或 Hugo）把 Markdown 文章编译成纯静态 HTML/CSS/JS， 再由 Cloudflare Pages 一键部署到全球边缘网络。优势包括：零服务器运维、极速首屏、按需扩缩、 免费额度充足。可扩展评论、搜索、RSS/邮件订阅、基础分析等功能。

推荐栈 A：Astro
  • 组件化 + Markdown/MDX；Tailwind 好用
  • 默认静态输出，CSR/SSR 按需启用
  • 生态：内容集合、RSS、站点地图插件
适合：现代审美与组件复用需求
推荐栈 B：Hugo
  • 编译速度极快，主题生态丰富
  • 多语言与内容组织能力强
  • 学习曲线低，文档齐全
适合：追求极简与快速出稿
托管与发布
  • Cloudflare Pages：自动构建&回滚
  • 自定义域名 + 自动化 HTTPS
  • 可选：KV/D1 做简单动态
成本：个人博客通常 0 元
一句话版部署流程：写 Markdown → 用 Astro/Hugo 生成静态文件 → 连接 Git 仓库 → Cloudflare Pages 自动构建 → 绑定域名 → 完成。

## 为什么选择静态博客？

- 极快：静态文件，天生适合 CDN。
- 安全：没有后端可以被攻击。
- 低成本：大多数场景免费。

> 小贴士：后续可接入评论系统与搜索。

站点结构与文件组织（建议）

清晰的目录结构让写作与维护更轻松。下面给出 Astro 与 Hugo 两种布局示例。

Astro 版结构
my-blog/ ├─ public/ │ ├─ images/ # 封面图、文章插图 │ └─ favicon.svg ├─ src/ │ ├─ content/ │ │ ├─ posts/ # Markdown/MDX 文章 │ │ └─ config.ts # 内容集合配置（可选） │ ├─ layouts/ │ │ ├─ Base.astro # 基础模板 │ │ └─ Post.astro # 文章页模板 │ ├─ components/ │ │ ├─ Header.astro │ │ ├─ Footer.astro │ │ ├─ PostCard.astro │ │ └─ GiscusComments.astro │ ├─ pages/ │ │ ├─ index.astro # 首页 │ │ ├─ posts/ │ │ │ └─ [slug].astro # 动态文章路由 │ │ ├─ rss.xml.ts # RSS │ │ └─ 404.astro # 自定义 404 │ └─ styles/ │ └─ global.css ├─ astro.config.mjs ├─ tailwind.config.js ├─ package.json └─ wrangler.json # 可选（用于 Pages Functions）
说明：
  • 文章放 src/content/posts，便于统一 frontmatter 与列表页
  • 组件化头部/底部与 PostCard 便于复用
  • 添加 rss.xml.ts 生成订阅源
Hugo 版结构
my-blog/ ├─ archetypes/ │ └─ default.md # 新文章默认 frontmatter ├─ assets/ # 可选：SCSS/图片处理 ├─ content/ │ ├─ _index.md # 首页内容 │ └─ posts/ │ ├─ first-post.md │ └─ second-post.md ├─ layouts/ │ ├─ _default/ │ │ ├─ baseof.html │ │ ├─ single.html │ │ └─ list.html │ └─ partials/ │ ├─ head.html │ ├─ header.html │ ├─ footer.html │ └─ comments.html ├─ static/ │ ├─ images/ │ └─ favicon.ico ├─ config.toml # 配置文件 └─ build.sh # 可选：本地构建脚本
说明：
  • Hugo 开箱即用，模板系统成熟
  • 用 archetypes 规范新文章字段
  • 图片放 static/images，引用 /images/...
文件命名建议：文章文件名用 kebab-case，如 why-static-blog.md； 日期可不放文件名，在 frontmatter 管理。统一标签（tags）与分类（categories）有助于归档与导航。

Markdown 写作规范与 Frontmatter

用约定好的字段，让列表页、详情页、RSS 与站点地图全自动生成。

推荐 Frontmatter 字段（Astro/Hugo 通用）
---
title: "文章标题"
description: "简短摘要，用于列表与社交卡片"
date: 2026-04-27              # 发布日期
updated: 2026-04-27           # 更新日期（可选）
tags: ["Cloudflare","Astro"]  # 标签
categories: ["Web"]           # 分类（可选）
cover: "/images/cover.jpg"    # 封面图
slug: "optional-slug"         # 自定义 URL（可选）
draft: false                  # 草稿标记
lang: "zh-CN"                 # 语言（多语言可选）
canonical: ""                 # 原文链接（转载时填写）
---
提示：为每个字段在列表与文章页显示对应元素（标签云、更新时间等）。
写作风格与媒体管理
  • 标题层级从 H2 开始，避免跳级
  • 段落控制在 3–5 行，列表化要点
  • 图片放 /images/...，使用相对路径
  • 代码块标注语言，采用行内注释说明关键点
  • 多媒体：优先静态图，视频用封面图 + 外链
Markdown 引用与表格示例：
> 这是一条引用，常用于提示或结论总结。

| 工具 | 作用 | 建议 |
| --- | --- | --- |
| Astro | 组件化 + MDX | 追求现代体验 |
| Hugo | 超快构建 | 重内容轻交互 |
图床与图片优化
  • 优先仓库内 /public/images 管理，也可用 Git LFS 或对象存储
  • 导出时用合适尺寸（首页 1200×630，文章内 1600px 宽度足够）
  • 文件名用英文短横分隔：cover-cloudflare-pages.jpg
  • 可启用 Cloudflare Image Resizing（可选进阶）
5

部署到 Cloudflare Pages

Pages 会监听你的 Git 仓库：每次推送 main 分支即自动构建与发布，并提供预览环境。

在 Cloudflare 控制台创建 Pages 项目

  1. 登录 Cloudflare → Workers & Pages → Create application → Pages → Connect to Git。
  2. 选择你的仓库与分支（通常是 main）。
  3. 选择预设（Astro），确认构建命令与输出路径：
Pages 构建配置（Astro 示例）
Framework preset: Astro
Build command:    npm ci && npm run build
Output directory: dist
Node version:     20.x（或 22.x）
Environment:      Production
Install command:  npm ci
提示：若构建失败，首先到 Deployments 查看日志，常见原因：Node 版本不匹配、缺少依赖、路径错误。

自定义环境变量（按需）

例如站点标题、分析 ID 等，可在 Pages 项目的 Settings → Environment variables 中添加。

# 示例：仅在生产环境生效（在 UI 中配置）
SITE_TITLE="我的博客"
UMAMI_WEBSITE_ID="xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"

（可选）wrangler.json 配置 Pages Functions/路由

若需要轻量动态能力（如 API）或自定义重写，可加入 wrangler.json。

wrangler.json 
{
  "$schema": "node_modules/wrangler/config-schema.json",
  "name": "my-blog",
  "compatibility_date": "2025-09-01",
  "pages_build_output_dir": "dist",
  "routes": [
    { "pattern": "/*", "zone_name": "你的域名（可选）" }
  ]
}

预览与回滚

  • 每次推送会生成 Preview 与 Production 环境
  • 可一键 Rollback 到上一个稳定部署
  • 可在 GitHub PR 开启自动预览链接
常见构建命令（参考）
# Astro
npm ci
npm run build   # 输出到 ./dist

# Hugo
hugo --gc --minify   # 输出到 ./public（在 Pages 设置里改为 public）

绑定自定义域名与 HTTPS

让博客通过你的域名访问，并自动启用 HTTPS。

在 Pages 绑定域名

  1. Pages 项目 → Custom domains → Set up a custom domain。
  2. 输入域名（如 blog.example.com 或 example.com）。
  3. 若域名在 Cloudflare 管理，按向导添加 CNAME 记录；否则在你的 DNS 商添加记录。
  4. 等待验证通过，系统自动部署证书。
DNS 记录示例
# 子域名形式（推荐）
blog.example.com   CNAME   your-project.pages.dev.

# 根域名（取决于 DNS 商是否支持 CNAME 扁平化或 ALIAS）
example.com        CNAME   your-project.pages.dev.   # 部分 DNS 支持
# 或使用 A/AAAA 记录（按 Pages 提供的目标）

安全与重定向

  • 启用 Always Use HTTPS（Cloudflare 自动）
  • 可开启 HSTS 增强安全性（谨慎，需确保全站 HTTPS）
  • 如需旧链接迁移，使用 _redirects 或 项目设置中的重定向
_redirects 示例（放在输出根目录）
# 从旧路径跳转到新路径
/old-path   /new-path   301!

# 强制 www 到非 www（或反之）
# www.example.com/*   https://example.com/:splat   301!
小贴士：绑定域名后，前往 Search Console、Bing Webmaster 提交站点与 sitemap。

常用功能增强：评论｜搜索｜订阅｜分析

在静态站点上用轻量外部服务补足动态能力。

评论系统（推荐：Giscus）

Giscus 基于 GitHub Discussions，零后端、开源、样式跟随站点。

在文章页插入（Astro 组件示例）
<!-- src/components/GiscusComments.astro -->
<section class="mt-10">
  <script
    src="https://giscus.app/client.js"
    data-repo="你的用户名/你的仓库"
    data-repo-id="你的仓库ID"
    data-category="General"
    data-category-id="分类ID"
    data-mapping="pathname"
    data-strict="0"
    data-reactions-enabled="1"
    data-emit-metadata="0"
    data-input-position="bottom"
    data-theme="dark"
    data-lang="zh-CN"
    crossorigin="anonymous"
    async>
  </script>
</section>
获取 repo-id 与 category-id：在 Giscus 官网按向导生成脚本即可。
替代方案
  • Utterances（更简单，但样式可定制性低）
  • Waline / Twikoo（需自建后端或使用它们的云服务）

站内搜索（Pagefind / Algolia）

Pagefind 完全静态、离线可用，适合个人博客；Algolia 体验最好，但有免费额度限制。

Pagefind 接入（构建后索引）
# 安装
npm i -D pagefind

# 在构建后运行索引（package.json）
"scripts": {
  "build": "astro build",
  "postbuild": "npx pagefind --site dist --glob \"**/*.html\""
}

# 在前端引入搜索 UI（示例放 public/pagefind/pagefind-ui.js）
<link href="/pagefind/pagefind-ui.css" rel="stylesheet">
<script src="/pagefind/pagefind-ui.js"></script>
<div id="search"></div>
<script>
  new PagefindUI({ element: "#search", showSubResults: true });
</script>

RSS 与订阅

生成 RSS/Atom，并在页脚显示订阅入口；可配合 Buttondown/ConvertKit 收集邮件。

Astro 生成 RSS（src/pages/rss.xml.ts）
import rss from '@astrojs/rss';
import { getCollection } from 'astro:content';

export async function GET(context) {
  const posts = await getCollection('posts', ({ data }) => !data.draft);
  posts.sort((a, b) => b.data.date - a.data.date);

  return rss({
    title: '我的博客',
    description: '技术与生活记录',
    site: context.site,
    items: posts.map((p) => ({
      title: p.data.title,
      pubDate: p.data.date,
      description: p.data.description,
      link: `/posts/${p.slug}/`,
      guid: `/posts/${p.slug}/`
    })),
    customData: `<language>zh-CN</language>`,
  });
}
在页脚加上 RSS 图标与链接：/rss.xml。

基础分析（Cloudflare Web Analytics / Umami）

隐私友好且不影响性能：可直接粘贴脚本到 <head>。

Cloudflare Web Analytics（示例）
<!-- 放在 head 结束前 -->
<script defer src="https://static.cloudflareinsights.com/beacon.min.js"
  data-cf-beacon='{"token": "YOUR_TOKEN"}'></script>
Umami（自托管或云服务）
<script defer src="https://your-umami-domain/umami.js"
  data-website-id="YOUR_WEBSITE_ID"></script>
组合建议：先上 Giscus + RSS，后续按需加 Pagefind 与分析脚本。保持静态为主，避免引入过多第三方脚本。

Cloudflare 缓存与规则

  • 静态资源（.css/.js/.png/.jpg/.svg）长缓存（1年）
  • HTML 适中缓存（几分钟到一小时），避免用户看到陈旧内容
  • 利用 Cache Everything（Pages 场景通常已优化）
  • 服务端重定向与 404 统一，避免软 404

可选：国内访问优化

Cloudflare 在国内直连时体验可能受跨境网络影响，可根据需求选择以下方案。

方案 A：轻度优化（推荐）

  • 减少第三方脚本与字体请求
  • 图片托管在同一域名，减少跨域
  • 开启 brotli/gzip（Cloudflare 默认）
  • 适当缩短 HTML 缓存时间，回源更快
优点：简单、无额外成本；缺点：对跨境链路问题改善有限。

方案 B：双栈发布（进阶）

  • 将静态产物同步到国内对象存储/托管（如对象存储 + CDN）
  • 使用二级域名区分：blog.example.com（全球），cn.example.com（国内）
  • 在文章中避免跨境依赖（字体、脚本、评论等）
  • 可将评论设置为仅全球域名展示
优点：国内访问更稳；缺点：需要额外部署与维护。
务实建议：先用方案 A 跑通与稳定内容，再按需评估是否上方案 B。保持构建与发布流程自动化，避免“双栈不一致”。

运维、备份与故障处理

常见问题（FAQ）

把最容易踩的坑先列出来。

我需要自己买服务器吗？

不需要。Cloudflare Pages 为静态站点提供托管与全球加速，个人博客通常在免费额度内。

构建总是失败怎么办？

优先看 Pages 的构建日志。常见原因：

  • Node 版本不一致：与本地保持一致（20.x 或 22.x）
  • 依赖未锁定：确保有 package-lock.json / pnpm-lock.yaml
  • 路径错误：输出目录应与 Pages 设置一致（Astro 是 dist，Hugo 是 public）
如何实现评论与搜索？

评论可用 Giscus（基于 GitHub Discussions），搜索可用 Pagefind（完全静态）或 Algolia（体验更好但需配额）。

国内访问不理想怎么办？

先减少第三方资源与字体，优化图片；若仍不满足，再考虑双栈发布（同步到国内 CDN）。本指南已提供两种方案与注意事项。

如何迁移老文章与链接？

导出为 Markdown，统一 frontmatter 与图片路径；用 _redirects 保留旧 URL，并在搜索控制台提交新的 sitemap。

我能用自己的域名吗？

可以。在 Pages 项目添加 Custom domain，按指引设置 CNAME（或 A/AAAA），系统会自动签发 HTTPS 证书。
